### 3.2 カテゴリ別の命名傾向

#### 攻撃ツール → 攻撃的・捕食者・恐ろしい名前が多い
- Hydra（多頭の蛇）、Medusa（石化の怪物）、BloodHound（猟犬）
- Metasploit（エクスプロイト）、Cobalt Strike（攻撃）
- John the Ripper（切り裂き犯）、Hashcat（引っ掻く）

#### 防御ツール → 守護・堅固・探知の名前が多い
- Vault（金庫）、Sentinel（歩哨）、Talisman（護符）
- Fortify（強化）、Harbor（港）、Hardhat（ヘルメット）
- SonarQube（音波探知）、Falco（隼）

#### 分析・検出ツール → 探索・発見・嗅覚のメタファーが多い
- TruffleHog（トリュフを探す豚）、Snort（鼻を鳴らす）
- Prowler（うろつく者）、ScoutSuite（偵察）
- Sherlock（探偵）、Slither（蛇のように這って探る）

#### Web3ツール → ファンタジー・神話色が強い
- Mythril（ミスリル）、Manticore（マンティコア）
- Medusa（メデューサ）、Echidna（ハリモグラ/怪物の母）
- Code4rena（闘技場）

### 3.3 記憶に残るセキュリティツール名の法則

#### 法則1: 強いメタファーを使う
- **TruffleHog**: 豚がトリュフ（シークレット）を嗅ぎ分けるイメージ → 即座にツールの機能が分かる
- **BloodHound**: 血の匂いを追う猟犬 → 攻撃経路を追跡するツールに完璧
- **Wireshark**: ネットワークを泳ぐサメ → パケットを捕食するイメージ

#### 法則2: 短くて発音しやすい
- Snyk（1音節）、Trivy（2音節）、Falco（2音節）、Nuclei（3音節）
- 長い名前は短縮される傾向（OWASP ZAP, ffuf）

#### 法則3: 文化的リファレンスを活用
- **Kali** Linux → 破壊の女神 = セキュリティテスト
- **Mythril** → トールキンの最強金属 = スマートコントラクト防御
- **Sherlock** → 名探偵 = バグ発見

#### 法則4: ギャップ（意外性）で印象付ける
- **Mimikatz**: 「かわいい猫」なのに凶悪なパスワード窃取ツール
- **Burp Suite**: 「げっぷ」という下品な名前なのに本格的なツール
- **Brownie**: お菓子/妖精の名前なのにスマートコントラクト開発ツール

#### 法則5: 動詞・行動を想起させる
- Slither（這う）、Snort（嗅ぐ）、Prowler（うろつく）、Fortify（強化する）
- ツールの動作を名前から想像できる

### 3.4 Web2 vs Web3 の命名文化の違い

| 観点 | Web2 | Web3 |
|------|------|------|
| **雰囲気** | 実用的・機能説明的 | ファンタジー・神話的 |
| **頻出パターン** | 合成語（機能+技術用語） | 神話・伝説の生き物 |
| **ブランド志向** | 企業名+製品名が多い | 独立した印象的な名前 |
| **遊び心** | 適度（Burp, TruffleHog等） | 強い（Mythril, Manticore等） |
| **コミュニティ文化** | オープンソース+エンタープライズ | ハッカソン/CTF文化と融合 |
| **名前の由来** | 英語が中心 | 多言語（ラテン語、ギリシャ語、他） |

### 3.5 命名パターン総合ランキング（記憶に残りやすさ）

1. **神話・ファンタジー由来** ★★★★★ — 強いイメージ、ストーリー性あり
2. **動物名メタファー** ★★★★★ — 直感的、ビジュアルが浮かぶ
3. **実在する英単語（意味転用）** ★★★★☆ — シンプルだが記憶に残る
4. **文化的リファレンス（人名等）** ★★★★☆ — 知っている人にはインパクト大
5. **造語（短い）** ★★★☆☆ — ブランド性は高いが意味が不明
6. **合成語（機能説明）** ★★★☆☆ — 分かりやすいが無味乾燥になりがち
7. **頭字語** ★★☆☆☆ — 覚えにくい、意味が伝わりにくい
8. **説明的（そのまま）** ★★☆☆☆ — 最も分かりやすいが印象に残らない
