# OpenClaw v2026.3.x Security Changelog

## Overview

The v2026.3.x series represents a major security hardening effort for OpenClaw, addressing **35 vulnerabilities** across 8 patch releases. This document provides a comprehensive overview of all security fixes in the v2026.3.x series.

**Release Period**: January 2026 - March 2026

**Total Fixes**: 35 security advisories

**Severity Distribution**:
- Critical: 0
- High: 7
- Medium: 28

## Version Summary

| Version | Release Date | Advisories Fixed | Highlights |

## v2026.3.8 (Latest)

### Exec Allowlist Bypass (2)

**GHSA-8g75-q649-6pv6** (Medium)
- **Summary**: system.run approvals did not bind mutable script operands across approval and execution
- **Impact**: Attacker could modify script after approval but before execution
- **Mitigation**: Bind script content hash at approval time

**GHSA-vhwf-4x96-vqx2** (Medium)
- **Category**: Path Traversal
- **Summary**: skills-install-download can be redirected outside the tools root by rebinding the validated base path
- **Impact**: Skill installation path traversal
- **Mitigation**: Lock base path during validation

### Authentication Bypass (1)

**GHSA-g7cr-9h7q-4qxq** (Medium)
- **Category**: MS Teams
- **Summary**: MS Teams sender allowlist bypass when route allowlist is configured and sender allowlist is empty
- **Impact**: Unauthorized MS Teams senders can access bot
- **Mitigation**: Fail-closed when sender allowlist is empty

## v2026.3.7

### Secret Leakage (1)

**GHSA-rchv-x836-w7xp** (High)
- **Summary**: Dashboard leaked gateway auth material via browser URL/query and localStorage
- **Impact**: Gateway credentials exposed in browser storage and URL
- **Mitigation**: Remove auth material from URL and localStorage

### SSRF (1)

**GHSA-6mgf-v5j7-45cr** (High)
- **Summary**: fetch-guard forwards custom authorization headers across cross-origin redirects
- **Impact**: Authorization header leakage via redirect chains
- **Mitigation**: Strip sensitive headers on cross-origin redirects

### Exec Allowlist Bypass (3)

**GHSA-j425-whc4-4jgc** (Medium)
- **Summary**: system.run env override filtering allowed dangerous helper-command pivots
- **Impact**: Environment variable manipulation to bypass exec allowlist
- **Mitigation**: Strict environment variable filtering

**GHSA-9q2p-vc84-2rwm** (Medium)
- **Summary**: system.run allow-always persistence included shell-commented payload tails
- **Impact**: Shell comment injection to hide malicious payloads
- **Mitigation**: Parse and validate entire command including comments

**GHSA-r6qf-8968-wj9q** (Medium)
- **Summary**: system.run wrapper-depth boundary could skip shell approval gating
- **Impact**: Nested wrapper commands bypass approval
- **Mitigation**: Enforce wrapper depth limits

**GHSA-3h2q-j2v4-6w5r** (Medium)
- **Summary**: system.run allowlist approval parsing missed PowerShell encoded-command wrappers
- **Impact**: PowerShell encoding bypass
- **Mitigation**: Detect and parse PowerShell encoded commands

### Authentication Bypass (1)

**GHSA-pjvx-rx66-r3fg** (Medium)
- **Summary**: Cross-account sender authorization expansion in /allowlist --store account scoping
- **Impact**: Cross-account authorization bypass
- **Mitigation**: Proper account scoping in allowlist storage

### Privilege Escalation (1)

**GHSA-hfpr-jhpq-x4rm** (Medium)
- **Summary**: operator.write chat.send could reach admin-only config writes
- **Impact**: Write-scope operators can modify admin config
- **Mitigation**: Strict permission checking for config writes

### Sandbox Escape (1)

**GHSA-9q36-67vc-rrwg** (Medium)
- **Summary**: Sandboxed /acp spawn requests could initialize host ACP sessions
- **Impact**: Sandbox escape via ACP initialization
- **Mitigation**: Enforce sandbox inheritance for ACP spawns

### DoS (1)

**GHSA-6rmx-gvvg-vh6j** (Medium)
- **Summary**: hooks count non-POST requests toward auth lockout
- **Impact**: DoS via auth lockout with non-POST requests
- **Mitigation**: Only count POST requests for lockout

## Upgrade Recommendations

### Immediate Action Required

**All deployments should upgrade to v2026.3.8 immediately.**

The v2026.3.x series addresses multiple **high-severity** vulnerabilities:
- **Sandbox escape** via ACP sessions_spawn
- **RCE** via working directory injection
- **SSRF** via DNS pinning loss
- **Secret leakage** of gateway credentials
- **Privilege escalation** in authorization checks

### Breaking Changes

See [Breaking Changes Guide](../best-practices/breaking-changes-guide.md) for detailed migration instructions.

**Major Changes**:
- **Sandbox Inheritance**: sessions_spawn now enforces sandbox for all cross-agent spawns
- **Exec Approval Binding**: system.run approvals now bind PATH and script content
- **Auth Fail-Closed**: Empty allowlists now deny access (previously allowed)

### Verification

After upgrading, verify the following:

1. **Sandbox Configuration**:
```bash
# Ensure sandbox is enabled
export OPENCLAW_SANDBOX=all
openclaw --version
```

2. **Test system.run Approvals**:
```bash
# Verify approval binds executable identity
# Attempt to modify PATH after approval - should fail
```

3. **Check Auth Configuration**:
```bash
# Verify all channels have explicit allowlists
openclaw channels audit --check empty-allowlist
```

4. **Review Gateway Security**:
```bash
# Ensure gateway auth material not in browser storage
# Check for GHSA-rchv-x836-w7xp
```
