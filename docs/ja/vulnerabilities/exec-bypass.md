# Exec Allowlist バイパス脆弱性

## 概要

OpenClawの `system.run` コマンド実行機構および `exec allowlist`（`safeBins`とも呼ばれる）は、ユーザーが承認したコマンドのみを実行させるための中核的セキュリティ機能です。しかし、v2026.1.x以降で **28件以上の固有なバイパス手法** が発見されており、OpenClawにおける最大かつ最も活発に悪用される攻撃面となっています（全セキュリティアドバイザリの約17%を占める）。

### 根本原因

シェルの複雑さ（bash、sh、zsh、cmd.exe、PowerShell、busybox等）に対して、**文字列ベースのパターンマッチングで防御することには根本的な限界があります**。チューリング完全なシェル言語に対してコマンドの同一性を静的に判定することは、理論的に困難です。

## 統計情報

| 項目 | 値 |
|------|-----|
| 脆弱性総数 | 28+ 手法（データベースには9件の正式GHSA登録） |
| 深刻度分布 | High: 3件、Medium: 6件 |
| 影響バージョン | v2026.1.x ~ v2026.3.7 |
| 最新修正バージョン | >= v2026.3.8 |
| 影響 | リモートコード実行（RCE） |

## 脆弱性の分類

### カテゴリ1: シェル構文バイパス（8手法）

シェルのパース処理の複雑さを悪用してallowlistチェックを回避する手法群。

| # | GHSA ID | 深刻度 | 手法 | 説明 |
|---|---------|--------|------|------|
| 1 | GHSA-3c6h-g97w-fg78 | **High** | GNUロングオプション省略 | GNU getoptが`--he`を`--help`に展開する仕様を悪用 |
| 2 | GHSA-3hcm-ggvf-rch5 | High | コマンド置換/バッククォート | ダブルクォート内の`$(...)`や`` `...` ``でコマンド実行 |
| 3 | GHSA-9868-vxmx-w862 | Medium | 行継続 + コマンド置換 | バックスラッシュ行継続とコマンド置換の組み合わせ |
| 4 | GHSA-9p38-94jf-hgjj | Medium | macOS引用符コマンド置換 | macOS固有の引用符処理の差異を悪用 |
| 5 | GHSA-6rcp-vxwf-3mfp | Medium | 位置引数キャリア | シェルラッパーの位置引数に隠しコマンドを埋め込み |
| 6 | GHSA-65rx-fvh6-r4h2 | High | heredoc展開 | 引用符なしheredocのシェル展開を悪用（CVE-2026-27209） |
| 7 | GHSA-qj77-c3c8-9c3q | High | Windows cmd.exeパース | Windows固有のコマンドパーサーの差異を悪用 |
| 8 | GHSA-3h2q-j2v4-6w5r | **Medium** | PowerShellエンコードコマンド | パーサーが`-EncodedCommand`ラッパーを見逃し |

**GHSA-3c6h-g97w-fg78**: GNUロングオプション省略によるExec allowlistバイパス
- **修正バージョン**: >= 2026.2.23
- **攻撃シナリオ**: 攻撃者が`mycmd --he`を提供すると、GNU getoptが`mycmd --help`に展開し、完全一致のみをチェックするallowlistを回避。
- **緩和策**: v2026.2.23以降にアップデート（allowlist照合前に完全なオプション展開を実行）。

**GHSA-3h2q-j2v4-6w5r**: PowerShellエンコードコマンドラッパーを承認パースが見逃し
- **修正バージョン**: >= 2026.3.7
- **攻撃シナリオ**: `powershell.exe -EncodedCommand <base64>`がallowlistパースをバイパス（Base64ペイロードをデコードしない）。
- **緩和策**: v2026.3.7以降にアップデート（PowerShellエンコードコマンドを承認前にデコード・検証）。

### カテゴリ3: ラッパー・ディスパッチチェーンバイパス（4手法）

コマンド実行のラッパーやディスパッチチェーンの検証不足を悪用する手法群。

| # | GHSA ID | 深刻度 | 手法 | 説明 |
|---|---------|--------|------|------|
| 17 | GHSA-48wf-g7cp-gr3m | Medium | env -Sラッパー | `env -S`でコマンドを間接実行 |
| 18 | GHSA-796m-2973-wc5q | **High** | env -Sラッパー解釈 | env -Sラッパーの解釈差異を悪用 |
| 19 | GHSA-gwqp-86q6-w47g | Medium | busybox/toybox sh -c | 軽量シェルの`sh -c`経由の間接実行 |
| 20 | GHSA-jj82-76v6-933r | Medium | ディスパッチチェーンのアンラップ不足 | env/shellディスパッチチェーンの展開が不十分 |

**GHSA-796m-2973-wc5q**: env -Sラッパー経由のsafeBinsバイパス
- **修正バージョン**: >= 2026.2.23
- **攻撃シナリオ**: `env -S "mycmd --forbidden-flag"`がallowlistをバイパス（envの`-S`（文字列分割）オプションでコマンドをラップ）。
- **緩和策**: v2026.2.23以降にアップデート（allowlistチェック前にenv -Sラッパーをアンラップ）。

### カテゴリ4: 承認メカニズムバイパス（5手法）

`system.run`の承認メカニズム自体を迂回する手法群。v2026.3.x系で集中的に発見。

| # | GHSA ID | 深刻度 | 手法 | 説明 |
|---|---------|--------|------|------|
| 21 | GHSA-hwpq-rrpf-pgcq | High | 承認ID不一致 | 表示されたコマンドと実際に実行されるバイナリが異なる |
| 22 | GHSA-q399-23r3-hfx4 | **High** | PATHトークン未バインド | 承認後に実行ファイルを差し替え可能（TOCTOU的） |
| 23 | GHSA-8g75-q649-6pv6 | **Medium** | スクリプトオペランド未バインド | 可変スクリプトオペランドが承認/実行間でバインドされず |
| 24 | GHSA-9q2p-vc84-2rwm | **Medium** | allow-always永続化の汚染 | シェルコメントペイロード末尾がallow-alwaysに永続化 |
| 25 | GHSA-r6qf-8968-wj9q | **Medium** | ラッパー深度境界スキップ | ラッパー深度制限がシェル承認ゲーティングをスキップ可能 |

**GHSA-q399-23r3-hfx4**: 承認時にPATHトークン実行ファイルIDをバインドしない
- **修正バージョン**: >= 2026.3.1
- **攻撃シナリオ**:
  1. ユーザーが`/usr/bin/mycmd`の実行を承認
  2. 攻撃者が実行前に`/usr/bin/mycmd`を悪意あるバイナリに差し替え
  3. 承認システムが実行ファイルIDの変更を検証しない
- **緩和策**: v2026.3.1以降にアップデート（承認と実行の間でPATHトークン実行ファイルIDをバインド）。

**GHSA-8g75-q649-6pv6**: 承認と実行の間でスクリプトオペランドをバインドしない
- **修正バージョン**: >= 2026.3.8
- **攻撃シナリオ**: 可変スクリプトオペランド（例: スクリプトファイルパス）を承認と実行の間で変更可能。
- **緩和策**: v2026.3.8以降にアップデート（可変スクリプトオペランドをバインド）。

**GHSA-9q2p-vc84-2rwm**: allow-always永続化にシェルコメントペイロード末尾を含む
- **修正バージョン**: >= 2026.3.7
- **攻撃シナリオ**: `mycmd # malicious-payload`がallow-alwaysに永続化され、将来のコンテキストでコメントペイロードが解釈される可能性。
- **緩和策**: v2026.3.7以降にアップデート（allow-always永続化をサニタイズ）。

**GHSA-r6qf-8968-wj9q**: ラッパー深度境界がシェル承認ゲーティングをスキップ可能
- **修正バージョン**: >= 2026.3.7
- **攻撃シナリオ**: ラッパー深度制限の実施をバイパスし、ネストされたラッパーが承認要件をスキップ可能。
- **緩和策**: v2026.3.7以降にアップデート（ラッパー深度制限を適切に実施）。

**GHSA-j425-whc4-4jgc**: env overrideフィルタリングが危険なヘルパーコマンドピボットを許可
- **修正バージョン**: >= 2026.3.7
- **攻撃シナリオ**: 環境変数オーバーライドフィルタリングが危険なヘルパーコマンドへのピボットを許可。
- **緩和策**: v2026.3.7以降にアップデート（env overrideフィルタリングを改善）。

## 攻撃パターンの進化タイムライン

```
v2026.1.x ~ v2026.2.26 (Phase 1-4: 初期発見)
├── シェル構文バイパス（GNUオプション、コマンド置換、heredoc等）
├── 環境変数インジェクション（PATH、BASH_ENV、SHELLOPTS等）
├── ラッパーチェーン迂回（env -S、busybox、ディスパッチ）
└── 既知の承認ID不一致

v2026.3.1 (Phase 5: 承認バインディング強化)
├── PATHトークン実行ファイルIDバインドを追加
└── 承認メカニズム自体への攻撃が本格化

v2026.3.2 (承認ハードニング)
└── セマンティックドリフトによるハードニングバイパスの発見

v2026.3.7 (Phase 6: 多角的バイパス)
├── env overrideピボット
├── allow-always永続化の汚染
├── ラッパー深度境界スキップ
└── PowerShellエンコードコマンドバイパス

v2026.3.8 (最新)
└── スクリプトオペランド未バインドの修正
```

## 影響評価

### 前提条件
- LLMエージェントが攻撃者の制御下にある（プロンプトインジェクション成功後）、または
- 悪意あるスキル/プラグインがインストールされている

### 影響
- **任意コマンド実行**: exec allowlistをバイパスして、許可されていないコマンドを実行
- **データ漏洩**: ファイルシステムアクセスによる認証情報の窃取
- **永続化**: allow-always機能を汚染し、以降の実行で承認なしにコマンド実行
- **横展開**: system.runから他のシステムコンポーネントへの攻撃ピボット

### 攻撃チェーンの例

```
プロンプトインジェクション
  └→ LLMがsystem.runを呼び出し
      └→ exec allowlistバイパス（例: env -S、heredoc展開）
          └→ 任意コマンド実行
              ├→ curlで外部C2にデータ送信
              ├→ ファイル読み取り（シークレット窃取）
              └→ allow-always汚染（永続化）
```

## 防御戦略

### 即時対応
1. **最新バージョンへアップデート**: v2026.3.8以降にアップグレードし、既知のバイパスの修正を適用
2. **exec allowlistの最小化**: 必要最小限のコマンドのみをallowlistに含める
3. **サンドボックスの有効化**: `sandbox.mode: all`を設定し、コマンド実行をコンテナ内に隔離

### 中期対応
4. **ランタイムチェックの実施**: `openclaw security audit --deep`でsystem.run設定を定期確認
5. **環境変数のサニタイズ**: 危険な環境変数（BASH_ENV、SHELLOPTS、PATH等）を制限
6. **allow-alwaysの定期監査**: 永続化された承認リストの内容を定期的にレビュー

### 長期対策
7. **アーキテクチャの再設計**: 文字列パターンマッチングから、実行バイナリのハッシュ検証・署名ベースの承認への移行
8. **サンドボックスファーストの実行モデル**: 全コマンド実行をデフォルトでサンドボックス内に限定
