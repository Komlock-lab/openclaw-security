# OpenClaw v2026.3.x セキュリティ変更ログ

## 概要

v2026.3.xシリーズは、OpenClawの大規模なセキュリティ強化を表し、8回のパッチリリースで**35件の脆弱性**に対処しています。このドキュメントは、v2026.3.xシリーズの全セキュリティ修正の包括的な概要を提供します。

**リリース期間**: 2026年1月 - 2026年3月

**修正総数**: 35件のセキュリティアドバイザリ

**深刻度分布**:
- Critical（最重要）: 0件
- High（高）: 7件
- Medium（中）: 28件

## バージョンサマリー

| バージョン | リリース日 | 修正アドバイザリ | ハイライト |
|---------|--------------|------------------|------------|
| v2026.3.8 | 2026-03-XX | 3件 | Exec承認バインディング、MS Teams認証バイパス |
| v2026.3.7 | 2026-03-XX | 7件 | Gateway認証漏洩、execラッパーバイパス |
| v2026.3.2 | 2026-03-XX | 11件 | ACPサンドボックス継承、TOCTOU修正 |
| v2026.3.1 | 2026-03-XX | 14件 | サンドボックススポーン、TOCTOU競合、認証バイパス |

## v2026.3.8（最新）

### Execホワイトリストバイパス（2件）

**GHSA-8g75-q649-6pv6**（Medium）
- **概要**: system.run承認が承認と実行の間で可変スクリプトオペランドをバインドしない
- **影響**: 攻撃者が承認後・実行前にスクリプトを変更可能
- **緩和策**: 承認時にスクリプトコンテンツハッシュをバインド

**GHSA-vhwf-4x96-vqx2**（Medium）
- **カテゴリ**: パストラバーサル
- **概要**: skills-install-downloadが検証済みベースパスのリバインドによりツールルート外にリダイレクト可能
- **影響**: スキルインストールのパストラバーサル
- **緩和策**: 検証中にベースパスをロック

### 認証バイパス（1件）

**GHSA-g7cr-9h7q-4qxq**（Medium）
- **カテゴリ**: MS Teams
- **概要**: ルートホワイトリスト設定時+送信者ホワイトリスト空でMS Teams送信者ホワイトリストバイパス
- **影響**: 未認可のMS Teams送信者がボットにアクセス可能
- **緩和策**: 送信者ホワイトリストが空の場合に失敗クローズ

## アップグレード推奨事項

### 即座の対応が必要

**全デプロイメントは即座にv2026.3.8にアップグレードすべきです。**

v2026.3.xシリーズは複数の**高深刻度**脆弱性に対処しています:
- ACP sessions_spawn経由の**サンドボックスエスケープ**
- 作業ディレクトリインジェクション経由の**RCE**
- DNSピンニング喪失経由の**SSRF**
- Gateway認証情報の**シークレット漏洩**
- 認可チェックでの**権限昇格**

### 破壊的変更

詳細な移行手順は [破壊的変更ガイド](../best-practices/breaking-changes-guide.md) を参照してください。

**主要な変更**:
- **サンドボックス継承**: sessions_spawnが全クロスエージェントスポーンでサンドボックスを強制
- **Exec承認バインディング**: system.run承認がPATHとスクリプトコンテンツをバインド
- **認証失敗クローズ**: 空のホワイトリストがアクセスを拒否（以前は許可）

### 検証

アップグレード後、以下を検証してください:

1. **サンドボックス設定**:
```bash
# サンドボックスが有効であることを確認
export OPENCLAW_SANDBOX=all
openclaw --version
```

2. **system.run承認をテスト**:
```bash
# 承認が実行可能ファイルIDをバインドすることを検証
# 承認後にPATHを変更を試行 - 失敗すべき
```

3. **認証設定をチェック**:
```bash
# 全チャンネルに明示的なホワイトリストがあることを検証
openclaw channels audit --check empty-allowlist
```

4. **Gatewayセキュリティをレビュー**:
```bash
# Gateway認証情報がブラウザストレージにないことを確認
# GHSA-rchv-x836-w7xpをチェック
```
